# ติดตั้งและอัปเดตระบบ Admin ข้อมูลสารเคมี

แพ็กเกจนี้ใช้ข้อมูลจากไฟล์:

`รายชื่อสารเคมีในโรงพยาบาลดอนตูม.xlsx`

## สรุปข้อมูลชุดใหม่

- รายการสารเคมี: **96 รายการ**
- หน่วยงาน/สถานะหน่วยงาน: **15 กลุ่ม**
- ชื่อสารเคมีไม่ซ้ำ: **96 ชื่อ**
- รายการที่ไม่ระบุหน่วยงานในไฟล์ต้นทาง: **19 รายการ**

---

## กรณียังไม่เคยติดตั้งระบบ Admin

1. สร้าง Google Sheets ใหม่
2. ไปที่ **ส่วนขยาย → Apps Script**
3. คัดลอกโค้ดจาก `apps-script/Code.gs` ไปวาง
4. ตั้ง Script Property:
   - Property: `ADMIN_PASSWORD`
   - Value: รหัสผ่าน Admin ที่ต้องการ
5. เลือกฟังก์ชัน `setup` แล้วกด **Run**
6. Deploy เป็น Web App:
   - Execute as: **Me**
   - Who has access: **Anyone**
7. คัดลอก URL ที่ลงท้าย `/exec`
8. แก้ `assets/config.js`:

```javascript
window.CHEMICAL_APP_CONFIG = {
  API_URL: "วาง URL /exec ที่นี่",
  API_ENABLED: true,
  PAGE_SIZE: 25
};
```

9. อัปโหลดไฟล์ทั้งหมดขึ้น GitHub
10. เปิด `admin.html` เข้าสู่ระบบ แล้วกด **นำเข้าข้อมูลตั้งต้น 96 รายการ**

## กรณีติดตั้งเวอร์ชันเดิมและมีข้อมูล 164 รายการแล้ว

### 1. อัปเดต Apps Script

1. เปิด Apps Script เดิม
2. แทนที่ `Code.gs` ด้วยไฟล์ `apps-script/Code.gs` ชุดใหม่นี้
3. กด **Deploy → Manage deployments**
4. กดรูปดินสอ
5. Version เลือก **New version**
6. กด **Deploy**

URL `/exec` เดิมยังใช้ได้

### 2. อัปโหลดเว็บไซต์ชุดใหม่

อัปโหลดไฟล์ทั้งหมดจาก ZIP นี้แทนไฟล์เดิมใน GitHub แล้วใช้ Commit message:

`เปลี่ยนข้อมูลสารเคมีเป็นไฟล์ฉบับใหม่`

### 3. แทนที่ข้อมูลเดิม

1. เปิด `admin.html`
2. เข้าสู่ระบบ Admin
3. กดปุ่มสีแดง **แทนที่ข้อมูลทั้งหมดด้วยไฟล์ใหม่ 96 รายการ**
4. อ่านคำเตือนและกดยืนยัน

ก่อนแทนที่ ระบบจะคัดลอกข้อมูลเดิมไปไว้ในชีตชื่อประมาณ:

`Backup_20260717_110000`

จึงสามารถกลับไปดูข้อมูลเดิมได้ภายหลัง

## การใช้งาน

- `index.html` สำหรับผู้ใช้ทั่วไป ดูและค้นหาข้อมูล
- `admin.html` สำหรับเพิ่ม แก้ไข ลบ และแทนที่ข้อมูล
- รายการที่หน่วยงานว่างใน Excel จะแสดงเป็น **ไม่ระบุหน่วยงาน**
- คอลัมน์ A ของไฟล์ใหม่ใช้เป็น **ลำดับ/รหัสรายการ**
- ชื่อสารเคมีอ่านจากคอลัมน์ D และใช้คอลัมน์ C เป็นข้อมูลสำรอง
- ปริมาณ: คอลัมน์ I
- สถานที่เก็บ: คอลัมน์ J
- การใช้ประโยชน์: คอลัมน์ K
- ความเป็นอันตราย: คอลัมน์ L–U
- PPE: คอลัมน์ V–Z

## ความปลอดภัย

- ห้ามใส่ข้อมูลผู้ป่วยหรือข้อมูลส่วนบุคคล
- ห้ามเขียนรหัสผ่านไว้ใน `assets/config.js`
- ตั้งรหัสผ่านเฉพาะใน Script Properties เท่านั้น
- หน้า Dashboard เป็นข้อมูลสาธารณะเมื่อเผยแพร่ผ่าน GitHub Pages
