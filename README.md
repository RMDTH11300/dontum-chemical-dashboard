# Dashboard ข้อมูลสารเคมี โรงพยาบาลดอนตูม

## แหล่งข้อมูลล่าสุด

- Excel: `WI-CHEM-001_Quick_Reference_สารเคมี_96_รายการ(3).xlsx`
- ภาพ: `สารเคมี (2).zip`

## สรุป

- สารเคมี: 96 รายการ
- หน่วยงาน: 21 หน่วยงาน
- กลุ่มอันตรายที่มีข้อมูล: 1 กลุ่ม
- ภาพ SDS: 96 ภาพ
- เวอร์ชันระบบ: 1.2.0
- ไม่มีระบบ Admin และไม่ใช้ Google Apps Script

## การจับคู่ภาพ

ระบบจับคู่ตามรหัสแบบ 1 ต่อ 1:

`CHEM-001 ↔ SDS-CHEM-001 ↔ chemical-001.webp`

จนถึง:

`CHEM-096 ↔ SDS-CHEM-096 ↔ chemical-096.webp`

## การปรับข้อมูลเพื่อการแสดงผล

- รวมการสะกด `กลุ่มงานปฐมภูมิ(งานประปา)` เป็น `กลุ่มงานปฐมภูมิ (งานประปา)`
- รวมคำสะกด `โภชนาการการ` เป็น `โภชนาการ`
- CHEM-009 มีคำว่า `กลุ่มงานปฐมภูมิ` อยู่ในช่องกลุ่มอันตราย ระบบย้ายไปแสดงเป็นหน่วยงาน

## วิธีอัปโหลด GitHub

1. แตกไฟล์ ZIP
2. อัปโหลดไฟล์ทั้งหมดแทนไฟล์เดิมใน Repository
3. Commit message:

`ปรับข้อมูลจาก Excel ฉบับ 3 และภาพ SDS ชุดใหม่`

4. รอ GitHub Pages อัปเดต แล้วกด `Ctrl + F5`


## รองรับโทรศัพท์มือถือ — Version 1.3.0

- เพิ่มปุ่ม **ตัวกรอง** และแผงตัวกรองแบบเลื่อนจากด้านซ้าย
- หน้าโทรศัพท์แสดงข้อมูลสารเคมีเป็นการ์ดแทนตารางกว้าง
- แสดงรายการครบทั้ง 96 รายการบนมือถือ
- ปุ่มและช่องกรอกมีขนาดเหมาะสำหรับการแตะ
- ปรับ KPI กราฟ รายการหน่วยงาน และหน้าเกี่ยวกับระบบให้พอดีจอ
- หน้าต่างดูภาพ SDS และรายละเอียดใช้พื้นที่เต็มหน้าจอโทรศัพท์
- รองรับทั้งโทรศัพท์แนวตั้งและแนวนอน


## อัปเดตภาพ SDS ชุดล่าสุด

ใช้ภาพจากไฟล์ `สารเคมี(2).zip` แทนภาพเดิมทั้งหมด

- ภาพครบ 96 รายการ
- จับคู่ตามรหัส CHEM-001 ถึง CHEM-096
- รองรับโทรศัพท์มือถือ
- เปิดภาพจากปุ่ม **ดูภาพ** เท่านั้น
- รายละเอียดการจับคู่อยู่ใน `data/image_mapping_latest.csv`


## รองรับโทรศัพท์และคอมพิวเตอร์ — Version 1.4.0

### โทรศัพท์และแท็บเล็ต (ความกว้างไม่เกิน 900px)
- แสดงสารเคมีเป็นการ์ด อ่านง่ายโดยไม่ต้องเลื่อนตารางแนวนอน
- ตัวกรองเปิดจากปุ่มด้านบนเป็นแผงเลื่อน
- หน้าดูภาพ SDS และรายละเอียดแสดงเต็มหน้าจอ
- ปุ่มมีขนาดเหมาะสำหรับการแตะ

### คอมพิวเตอร์และโน้ตบุ๊ก (มากกว่า 900px)
- แสดง Sidebar ตัวกรองตลอดเวลา
- แสดง Dashboard กราฟ และตารางทะเบียนครบ 11 คอลัมน์
- ตารางเลื่อนแนวนอนอยู่ภายในกรอบ ไม่ทำให้หน้าเว็บล้น
- หน้าดูภาพ SDS แสดงเป็นหน้าต่างขนาดเหมาะสม

แก้ไข breakpoint ของ JavaScript และ CSS ให้ตรงกันที่ 900px
เพื่อป้องกันช่วงหน้าจอแท็บเล็ตแสดงข้อมูลไม่ครบ
